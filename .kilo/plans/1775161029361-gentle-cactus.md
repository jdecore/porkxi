# Análisis del Proyecto porkxi

## 1. ¿Por qué hay dos carpetas?

| Carpeta | Propósito | Estado |
|---------|-----------|--------|
| `porkxi/` (raíz) | Proyecto original React 18 + Recharts + Vite | **Obsoleto** - fue migrado |
| `porkxi-astro/` | Migración a Astro 6 (HTML estático puro) | **Activo** |

La carpeta raíz contiene el proyecto React original (~150KB de output). `porkxi-astro/` es la versión migrada que genera solo **64KB con 0KB de JavaScript**.

## 2. Estado actual de Astro en porkxi-astro

### Lo que funciona bien
- **0KB JavaScript** en output — todo es HTML+CSS estático
- Gráfico SVG calculado en build time dentro de `GraficaPrincipal.astro`
- Todo el contenido renderizado en `src/pages/index.astro` con lógica Astro (`.map()`, `.filter()`)
- Sin hidratación de ningún tipo

### Problemas detectados
1. **Dependencias huérfanas**: `package.json` incluye `@astrojs/vue` y `vue` pero `astro.config.mjs` **NO tiene la integración Vue configurada**:
   ```js
   // astro.config.mjs actual — falta la integración Vue
   export default defineConfig({ build: { inlineStylesheets: 'auto' } });
   ```

2. **Componentes Vue sin usar**: Existen 9 archivos `.vue` en `src/components/` que están **huérfanos**:
   - `BannerAlerta.vue`, `ComparativaFrecuencia.vue`, `DetalleUltimoReporte.vue`
   - `Encabezado.vue`, `GraficaPrincipal.vue`, `PieDePagina.vue`
   - `TablaHistorica.vue`, `TarjetasKpi.vue`
   
   Ninguno se importa en `index.astro` — todo el markup está inline.

3. **recharts como dependencia inútil**: Está en `package.json` pero no se usa (el gráfico es SVG manual).

4. **AGENTS.md contradictorio**: Dice "NO usar Vue ni React" pero las dependencias de Vue están instaladas.

## 3. Sobre "disminuir JavaScript y usar más Vue 3"

### Realidad importante
**Agregar Vue 3 AUMENTA el JavaScript, no lo disminuye.**

| Enfoque | JS Output | Interactividad |
|---------|-----------|----------------|
| Astro puro (actual) | **0KB** | Ninguna (estático) |
| Vue con `client:load` | ~40-60KB | Inmediata al cargar |
| Vue con `client:visible` | ~40-60KB | Al hacer scroll |
| Vue con `client:idle` | ~40-60KB | Cuando el navegador está libre |

### Opciones disponibles

#### Opción A: Mantener Astro puro (recomendada para SEO + velocidad)
- **JS**: 0KB
- **Ventaja**: Más rápido, mejor SEO, funciona sin JS
- **Desventaja**: Sin interactividad (no hay tooltips en gráfica, no hay animaciones)
- **Acción**: Limpiar dependencias huérfanas (`vue`, `@astrojs/vue`, `recharts`)

#### Opción B: Usar Vue 3 selectivamente con hidratación parcial
- **JS**: ~40-60KB (solo en componentes que lo necesiten)
- **Ventaja**: Interactividad donde importa (tooltips en gráfica, tabs, etc.)
- **Desventaja**: Aumenta el bundle
- **Estrategia**: Usar `client:visible` para cargar Vue solo cuando el componente entra en viewport

#### Opción C: Vue 3 islands (híbrido)
- Mantener todo el contenido estático en Astro
- Usar Vue **solo** para la gráfica con interactividad (tooltips al hover)
- **JS estimado**: ~25KB con tree-shaking

## 4. Plan de acción: Mantener Astro puro (0KB JS)
